# 贡献指南 (Contributing Guide)

感谢你对 **AI-Powered Display Handbook** 感兴趣！这是一个旨在帮助所有人读懂显示器参数的开源项目。我们非常欢迎社区的贡献，无论是纠正错别字、补充新知识，还是分享你的实测经验。

## 如何参与贡献？

### 1. 提交 Issue (问题反馈)
如果你发现了文档中的错误，或者有好的建议（例如希望增加某个新技术的解析），请直接提交 Issue。
*   **Bug**: 指出具体的错误位置和正确内容。
*   **Feature**: 描述你希望增加的内容及其价值。

### 2. 提交 Pull Request (代码合并)
如果你愿意直接动手完善文档，请遵循以下步骤：

1.  **Fork 本仓库**：点击右上角的 Fork 按钮。
2.  **创建分支**：`git checkout -b my-new-feature`
3.  **修改内容**：
    *   请保持 Markdown 格式整洁。
    *   如果是新增章节，请确保在 `mkdocs.yml` 中添加导航。
    *   引用数据请尽量注明来源。
4.  **提交更改**：`git commit -m 'Add some feature'`
5.  **推送到分支**：`git push origin my-new-feature`
6.  **发起 Pull Request**：在 GitHub 页面点击 "New Pull Request"。

## 文档规范

*   **语气**：保持客观、专业，但通俗易懂。避免过于晦涩的学术用语。
*   **排版**：
    *   使用 `!!! tip` 等 Admonition 语法来高亮关键信息。
    *   专业术语（如 G-Sync, HDR）前后请加空格。
*   **图片**：尽量使用 Mermaid 绘制图表。如果必须使用图片，请确保无版权争议，并放入 `docs/assets/images` 目录。

## 行为准则

请友善交流，尊重不同的观点。我们致力于构建一个开放、包容的技术社区。
